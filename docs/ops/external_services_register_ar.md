\*\*تصنيف الوثيقة:\*\* سجل تشغيلي داخلي  

\*\*البريد الرسمي:\*\* info@diwaniya.online  

\*\*الدومين الرسمي:\*\* diwaniya.online  

\*\*API الإنتاج:\*\* https://api.diwaniya.online  



\---



\## 1. الهدف من السجل



يهدف هذا السجل إلى توثيق جميع الخدمات والاشتراكات الخارجية المستخدمة في تشغيل تطبيق ديوانية، مع توضيح الغرض، التكلفة، مستوى الحساسية، البيانات المعالجة، ومتطلبات الأمان والخصوصية.



يُستخدم هذا الملف لضبط التكاليف، منع التكرار، توثيق الاعتمادات الخارجية، وتسهيل المراجعة التشغيلية والقانونية والأمنية قبل الإطلاق وبعده.



\---



\## 2. مبادئ إدارة الخدمات الخارجية



يعتمد تطبيق ديوانية المبادئ التالية:



1\. عدم استخدام أي خدمة خارجية إلا لغرض واضح ومبرر.

2\. تقليل عدد الخدمات والاشتراكات لتقليل التكلفة والتعقيد.

3\. عدم تمرير بيانات شخصية أو محتوى خاص إلى أي مزود دون حاجة تشغيلية أو أساس نظامي مناسب.

4\. مراجعة أي SDK أو مزود جديد قبل إضافته إلى التطبيق أو الباكند.

5\. توثيق التكلفة والمالك والغرض لكل خدمة.

6\. تجنب الخدمات التي تسبب vendor lock-in عالي دون ضرورة.

7\. الفصل بين بيئة الإنتاج وبيئة الاختبار قدر الإمكان.

8\. عدم استخدام رموز اختبار أو بيانات تجريبية في الإنتاج.

9\. مراجعة الأمان والخصوصية لكل مزود يحتوي على بيانات مستخدمين.

10\. تحديث هذا السجل عند إضافة أو حذف أو تغيير أي خدمة.

\## 4. تفاصيل الخدمات



\## 4.1 Namecheap



| البند | التفاصيل |

|---|---|

| الغرض | تسجيل الدومين diwaniya.online |

| الحالة | نشط |

| الدومين | diwaniya.online |

| الاستخدام الحالي | تسجيل الدومين فقط |

| DNS الحالي | مدار عبر Cloudflare |

| يحتوي بيانات مستخدمين؟ | لا مباشرة |

| مستوى الحساسية | منخفض |

| ملاحظات | لا نحتاج استضافة Namecheap حاليًا. لا يتم تعديل DNS من Namecheap بعد نقل Nameservers إلى Cloudflare. |



\### ضوابط Namecheap



\- تفعيل 2FA للحساب.

\- عدم شراء استضافة إضافية دون حاجة.

\- تجديد الدومين قبل انتهائه.

\- توثيق تاريخ التجديد والتكلفة لاحقًا.



\---



\## 4.2 Cloudflare



| البند | التفاصيل |

|---|---|

| الغرض | DNS، HTTPS، حماية أساسية، استضافة الموقع العام عبر Pages/Workers |

| الحالة | نشط |

| الدومينات | diwaniya.online، www.diwaniya.online، api.diwaniya.online |

| الموقع العام | https://diwaniya.online |

| حذف الحساب | https://diwaniya.online/delete-account |

| API DNS | api.diwaniya.online يشير إلى Render |

| التكلفة | مجانية حاليًا |

| يحتوي بيانات مستخدمين؟ | لا مباشرة للموقع العام، وقد يرى سجلات طلبات DNS/HTTP |

| مستوى الحساسية | متوسط |



\### ضوابط Cloudflare



\- تفعيل 2FA.

\- جعل سجلات البريد DNS only.

\- جعل api.diwaniya.online DNS only عند ربطه بـ Render.

\- عدم حذف سجلات MX/TXT/DKIM/SPF الخاصة بالبريد.

\- تفعيل Always Use HTTPS للموقع العام إن أمكن.

\- عدم استخدام روابط workers.dev أو pages.dev في المتاجر.

\- مراجعة أي Worker أو Route قبل تفعيله على الدومين.

\## 4.3 Render



| البند | التفاصيل |

|---|---|

| الغرض | تشغيل FastAPI Backend وAPI الإنتاج |

| الحالة | نشط |

| الخدمة | diwaniya-backend |

| رابط Render | https://diwaniya-backend.onrender.com |

| رابط API الرسمي | https://api.diwaniya.online |

| التكلفة | تقريبًا 10 دولار شهريًا |

| يحتوي بيانات مستخدمين؟ | نعم |

| مستوى الحساسية | عالٍ |



\### Environment Variables غير السرية المعتمدة



| المتغير | القيمة الإنتاجية المطلوبة |

|---|---|

| APP\_ENV | production |

| APP\_DEBUG | false |

| ALLOW\_TEST\_OTP | false |

| ALLOWED\_ORIGINS | \[\\"https://diwaniya.online\\",\\"https://www.diwaniya.online\\"] |



\### Secrets موجودة ولا توثق قيمها



\- DATABASE\_URL

\- JWT\_SECRET

\- ADMIN\_JWT\_SECRET

\- أي مفاتيح OTP لاحقًا



\### ضوابط Render



\- عدم مشاركة قيم الأسرار في المحادثات أو Git.

\- تعطيل OTP التجريبي في الإنتاج.

\- استخدام HTTPS فقط.

\- إبقاء health/healthz/readyz مفعلة.

\- مراقبة deploy logs بعد كل نشر.

\- عدم تفعيل auto-deploy دون مراجعة عند الاقتراب من الإطلاق النهائي، أو إبقاؤه مع branch محمي.

\- توثيق أي فشل نشر وأسبابه.

\## 4.8 OTP Provider



| البند | التفاصيل |

|---|---|

| الغرض | التحقق من رقم الجوال |

| الحالة | لم يحدد |

| يحتوي بيانات مستخدمين؟ | نعم، أرقام جوال ومحاولات تحقق |

| مستوى الحساسية | عالٍ |

| الخيارات | SMS، WhatsApp OTP، أو مزيج بينهما |



\### معايير اختيار مزود OTP



1\. تكلفة الرسالة داخل السعودية.

2\. دعم WhatsApp OTP إن كان أقل تكلفة وأفضل وصولًا.

3\. دعم SMS fallback.

4\. سهولة الربط مع FastAPI.

5\. تقارير التسليم.

6\. Rate limiting.

7\. خصوصية البيانات.

8\. قابلية التوسع.

9\. عدم الحاجة لتعقيد كبير في البداية.

10\. دعم عربي وتجربة مستخدم جيدة.



\### ضوابط OTP



\- عدم استخدام TEST OTP في الإنتاج.

\- تفعيل rate limiting.

\- تسجيل محاولات فاشلة.

\- منع spam OTP.

\- إخفاء تفاصيل الفشل عن المستخدم.

\- عدم طباعة OTP في production logs.

\## 5. الخدمات غير المطلوبة حاليًا



لا نحتاج حاليًا:



1\. استضافة Namecheap.

2\. PremiumDNS من Namecheap.

3\. CDN مدفوع.

4\. SDK إعلاني.

5\. Payment gateway.

6\. Marketplace provider.

7\. AI production provider داخل التطبيق.

8\. أدوات تحليلات ثقيلة.

\## 7. مخاطر الخدمات الخارجية



| الخطر | المستوى | الإجراء |

|---|---|---|

| OTP غير مربوط | عالٍ | اختيار مزود قبل الإطلاق العام |

| Render يحتوي بيانات مستخدمين | عالٍ | ضبط secrets وlogs وaccess |

| GitHub قد يحتوي أسرار بالغلط | عالٍ | secret scanning و.gitignore |

| البريد يستقبل طلبات خصوصية | متوسط | تنظيم التصنيفات والردود |

| Cloudflare DNS خطأ قد يعطل الخدمة | متوسط | توثيق السجلات وعدم تعديلها عشوائيًا |

| خدمات جديدة دون مراجعة | عالٍ | مراجعة امتثال قبل إضافة أي SDK |

\## 8. إجراءات قبل الإطلاق



1\. اختيار OTP Provider.

2\. مراجعة Render secrets.

3\. مراجعة GitHub secrets.

4\. تشغيل فحص عدم وجود .env في المستودعات.

5\. تفعيل 2FA لجميع الحسابات.

6\. توثيق تكلفة كل خدمة.

7\. تحديد مالك كل حساب.

8\. تحديث هذه الوثيقة بعد أي تغيير.

9\. ربط أي خدمة جديدة بمراجعة خصوصية وأمن.

10\. اعتماد هذه الوثيقة ضمن حزمة التشغيل.

## 1. الهدف من الوثيقة

تهدف هذه الوثيقة إلى تجهيز تطبيق **ديوانية** لمتطلبات النشر والتشغيل في متجري Apple App Store وGoogle Play، وتوحيد الموقف التشغيلي والقانوني للتطبيق فيما يتعلق بالخصوصية، البيانات، الإعلانات، التسويق، مشاركة البيانات، صلاحيات الجهاز، حذف الحساب، المحتوى، مزايا التطبيق، وخدماته الحالية أو المستقبلية.

تُستخدم هذه الوثيقة كمرجع داخلي عند تعبئة نماذج المتاجر، إعداد وصف التطبيق، مراجعة السياسات، تشغيل مزودي الخدمة، تقييم المزايا الجديدة، أو إطلاق أي معالجة إضافية للبيانات. ولا تُعد هذه الوثيقة بديلًا عن الاستشارة القانونية النهائية، لكنها تُعد مرجعًا عمليًا لتقليل التعارض بين سلوك التطبيق الفعلي، السياسات المنشورة، متطلبات المتاجر، والأنظمة ذات العلاقة في المملكة العربية السعودية.

## 2. المبادئ الحاكمة للامتثال

يعتمد تطبيق ديوانية المبادئ التالية في التشغيل والإطلاق:

1. الشفافية في توضيح أنواع البيانات التي يتم جمعها أو استخدامها أو مشاركتها.
2. جمع البيانات بالقدر اللازم لتشغيل التطبيق، تقديم مزاياه، تحسينه، حمايته، والوفاء بالالتزامات النظامية أو التعاقدية.
3. عدم استخدام البيانات الشخصية أو مشاركتها إلا عند وجود أساس نظامي صحيح، أو موافقة عند الحاجة، أو مصلحة مشروعة، أو حاجة تشغيلية، أو التزام نظامي.
4. عزل بيانات كل ديوانية عن غيرها، وعدم إظهارها إلا للمستخدمين المخولين بحسب العضوية والصلاحيات.
5. ربط السجلات الحساسة أو التنظيمية بمعرفات داخلية ثابتة متى كان ذلك لازمًا لحماية السجلات ومنع الالتباس، بدل الاعتماد على بيانات قابلة للتغيير مثل الاسم أو رقم الجوال.
6. تمكين المستخدم من طلب حذف الحساب والبيانات، مع مراعاة السجلات الجماعية، الأرصدة، الالتزامات القائمة، وحماية حقوق المستخدمين الآخرين.
7. الإفصاح للمتاجر عن البيانات والمزايا والصلاحيات بطريقة تعكس سلوك التطبيق الفعلي.
8. مراجعة أي ميزة أو تكامل أو مزود خدمة جديد قد يؤثر على البيانات أو التسويق أو الإعلانات أو المشاركة أو التخزين قبل إطلاقه.
9. استخدام لغة واضحة ومفهومة للمستخدم، مع الحفاظ على صياغة رسمية راقية ومتوافقة مع هوية التطبيق.

## 3. حالة الامتثال الحالية

### 3.1 الوثائق القانونية الجاهزة داخل التطبيق

تم تجهيز الوثائق التالية داخل التطبيق ضمن شاشة واحدة باسم **الخصوصية والشروط**:

1. سياسة الخصوصية.
2. شروط الاستخدام.
3. سياسة الاستخدام والمحتوى.
4. سياسة الاحتفاظ بالبيانات وحذفها.

### 3.2 الوثائق الداخلية الجاهزة

تم تجهيز الوثائق الداخلية التالية:

1. خطة التعامل مع حوادث أمن المعلومات وتسرب البيانات.
2. حزمة جاهزية المتاجر والامتثال.

### 3.3 حذف الحساب والبيانات

يدعم التطبيق مسارًا داخل التطبيق لطلب حذف الحساب، مع مراعاة:

* منع حذف الحساب إذا كان للمستخدم رصيد أو التزام قائم داخل أي ديوانية.
* منع حذف الحساب إذا كان المستخدم يملك دورًا إداريًا لازمًا لاستمرارية ديوانية تضم أعضاء آخرين، إلى أن تتم معالجة ذلك الدور.
* توضيح أن بعض السجلات الجماعية قد تبقى مفصولة عن هوية المستخدم عند الحاجة لحماية حقوق أعضاء آخرين أو سلامة السجلات.
* عدم تسجيل خروج المستخدم عند فشل طلب الحذف.
* تسجيل الخروج فقط عند نجاح طلب الحذف.

### 3.4 شرط الرصيد أو الالتزام القائم

تم اعتماد قاعدة تشغيلية وتقنية مفادها:

* لا يمكن إزالة عضو من ديوانية إذا كان له أو عليه رصيد أو التزام قائم في تلك الديوانية.
* لا يمكن للعضو مغادرة ديوانية إذا كان له أو عليه رصيد أو التزام قائم في تلك الديوانية.
* لا يمكن حذف الحساب إذا كان للمستخدم رصيد أو التزام قائم داخل أي ديوانية.
* السجلات التنظيمية المرتبطة بالمستخدم تُربط بمعرف المستخدم الداخلي متى كان ذلك لازمًا، وليس بالاسم أو رقم الجوال فقط.
* أي رصيد أو التزام يكون محصورًا في نطاق الديوانية المعنية، ولا يمنع العضو من إجراءات تخص ديوانية أخرى لا يوجد له فيها رصيد أو التزام.

## 4. موقف التطبيق من الإعلانات والتسويق ومشاركة البيانات

### 4.1 القرار المستدام

قد يستخدم تطبيق ديوانية بيانات محدودة، عامة، مجمعة، إحصائية، أو غير معرّفة قدر الإمكان لأغراض تشغيلية أو تحليلية أو تسويقية أو إعلانية، بما يتوافق مع الأنظمة المعمول بها، وسياسات المتاجر، وسياسة الخصوصية، والموافقات المطلوبة عند انطباقها.

ولا يعني ذلك بيع البيانات الشخصية المباشرة أو إتاحة بيانات المستخدمين الخام لأطراف غير مخولة. أي استخدام تسويقي أو إعلاني يجب أن يلتزم بما يلي:

1. عدم مشاركة محتوى المستخدمين الخاص أو بيانات الديوانيات الخاصة مع المعلنين أو الشركاء التسويقيين إلا وفق أساس نظامي صحيح ومتى كان ذلك مفصحًا عنه ومسموحًا نظامًا.
2. عدم بيع البيانات الشخصية المباشرة للمستخدمين.
3. استخدام بيانات عامة أو مجمعة أو إحصائية أو غير معرفة قدر الإمكان.
4. الحصول على الموافقات اللازمة عند استخدام بيانات شخصية لأغراض تتطلب موافقة.
5. الإفصاح بوضوح داخل سياسة الخصوصية ونماذج Apple وGoogle عند تفعيل أي مزود خدمة أو SDK أو شريك تحليلي أو إعلاني.
6. تمكين المستخدم من إدارة الموافقات أو إعدادات الخصوصية عند الحاجة.
7. عدم استخدام البيانات الحساسة أو بيانات الفئات التي تتطلب حماية خاصة لأغراض تسويق أو إعلانات مخصصة إلا وفق ما تسمح به الأنظمة وبالموافقات المطلوبة.

### 4.2 الإعلانات والعروض والرعايات داخل التطبيق

قد يدعم التطبيق إعلانات أو عروضًا أو رعايات أو رسائل ترويجية أو محتوى تجاريًا مرتبطًا بسياق الخدمة أو الموقع العام أو الاهتمامات العامة أو مزايا التطبيق، بشرط ألا يؤدي ذلك إلى كشف محتوى الديوانيات أو بيانات أعضائها الخاصة للمعلنين أو الشركاء غير المخولين.

يجوز استخدام مؤشرات عامة ومجمعة مثل:

* مؤشرات الاستخدام العامة على مستوى مدينة أو منطقة أو فئة عامة دون تحديد المستخدمين.
* أداء الحملات داخل التطبيق بطريقة إحصائية.
* عدد مرات ظهور إعلان أو تفاعل عام معه.
* مؤشرات تشغيلية تساعد على تحسين الخدمة أو تقييم جدوى المزايا.

ولا يجوز للمعلنين أو الشركاء غير المخولين الاطلاع على:

* محتوى الديوانيات الخاصة.
* قوائم الأعضاء داخل ديوانية محددة.
* الرسائل أو الصور أو الملفات أو الصوتيات الخاصة.
* أرقام الجوال أو معرفات المستخدمين الداخلية.
* السجلات التفصيلية أو الأرصدة أو الالتزامات الخاصة بالأعضاء.

### 4.3 التسويق والتواصل

قد يستخدم التطبيق قنوات التواصل المتاحة، مثل رقم الجوال أو الإشعارات أو البريد عند توفره، لإرسال رسائل تشغيلية أو تنبيهات أو تحديثات مهمة أو عروض أو رسائل تسويقية، وفق الأنظمة والموافقات المطلوبة وخيارات المستخدم.

يجب التمييز بين:

* الرسائل التشغيلية اللازمة للخدمة، مثل التحقق، تنبيهات الأمان، تحديثات الحساب، طلبات العضوية، أو التنبيهات المهمة.
* الرسائل التسويقية أو الترويجية التي قد تتطلب موافقة أو خيار إلغاء الاشتراك.

### 4.4 مشاركة البيانات مع مزودي الخدمة

يجوز مشاركة بيانات محدودة مع مزودي خدمات يساعدون في تشغيل التطبيق أو تحسينه أو حمايته أو تقديم مزاياه، مثل:

* الاستضافة السحابية.
* قواعد البيانات.
* التخزين.
* التحقق وتسجيل الدخول.
* الإشعارات.
* مراقبة الأخطاء والأداء.
* التحليلات التشغيلية.
* الحماية والأمان.
* الدعم الفني.
* الاشتراكات أو المدفوعات عند تفعيلها.
* التسويق أو الإعلان أو القياس عند تفعيلها.
* أي مزود خدمة آخر لازم لتشغيل ميزة من مزايا التطبيق.

يجب أن يقتصر استخدام مزودي الخدمة للبيانات على الغرض المتفق عليه، وبالقدر اللازم، ووفق التزامات السرية والأمان المناسبة.

## 5. أنواع البيانات التي يعالجها التطبيق

### 5.1 بيانات الحساب والتحقق

قد تشمل:

* رقم الجوال.
* الاسم الأول أو الاسم المعروض.
* معرف المستخدم الداخلي.
* حالة الحساب.
* بيانات التحقق ومحاولات التحقق.
* تاريخ إنشاء الحساب وآخر استخدام.

الأغراض:

* إنشاء الحساب.
* تسجيل الدخول.
* التحقق من الهوية.
* حماية الحساب.
* منع إساءة الاستخدام.
* دعم حذف الحساب أو طلبات الخصوصية.

### 5.2 بيانات الديوانية والعضوية

قد تشمل:

* اسم الديوانية.
* بيانات الموقع العام التي يدخلها المستخدم، مثل الدولة أو المدينة أو الحي عند توفرها.
* رمز الدعوة.
* المؤسس والمدراء والأعضاء.
* حالة العضوية والدور.
* طلبات الانضمام وحالاتها.
* سجلات الترقية أو التنزيل أو الإزالة.

الأغراض:

* إنشاء وإدارة الديوانيات.
* إدارة العضويات والصلاحيات.
* حماية كل ديوانية وعزل بياناتها.
* تشغيل التنبيهات والنشاطات.

### 5.3 محتوى المستخدمين داخل التطبيق

قد يشمل محتوى المستخدمين جميع البيانات أو المواد أو السجلات التي يضيفها المستخدم أو ينشئها أو يتفاعل معها داخل خصائص التطبيق، بما في ذلك على سبيل المثال لا الحصر:

* النصوص والرسائل والتعليقات والردود.
* الصور والملفات والصوتيات والمرفقات.
* البيانات التنظيمية أو الجماعية أو التفاعلية داخل الديوانية.
* عناصر أو سجلات أو طلبات أو عروض أو محتوى يتم إنشاؤه داخل مزايا التطبيق الحالية أو المستقبلية.
* أي بيانات ترتبط بتفاعل المستخدم مع خصائص التطبيق أو أعضاء الديوانية.

الأغراض:

* تمكين التواصل والتنظيم داخل الديوانية.
* إدارة النشاط الجماعي بين الأعضاء.
* تشغيل الخصائص الأساسية والمستقبلية.
* تحسين تجربة المستخدم.
* حماية الخدمة ومنع الإساءة.

### 5.4 البيانات التنظيمية والالتزامات المشتركة

قد تشمل البيانات التنظيمية أو الالتزامات المشتركة أي سجلات ينشئها المستخدمون داخل الديوانية لإدارة شؤونهم، مثل السجلات المشتركة أو الأرصدة أو الالتزامات أو المتابعات أو القرارات أو أي بيانات تنظيمية مشابهة.

ملاحظة مهمة:

أي سجلات تنظيمية أو أرصدة داخل التطبيق تُعد أدوات تنظيمية بين أعضاء الديوانية، ولا تمثل خدمة دفع أو تحويل مالي أو تحصيل أو ضمان أو وساطة مالية، ما لم يتم تفعيل خدمة صريحة مستقلة وفق شروط وإفصاحات إضافية.

### 5.5 بيانات المزايا التجارية أو التفاعلية

قد تشمل عند توفرها:

* بيانات العروض أو القوائم أو الخدمات أو المنتجات أو المحتوى التجاري أو التفاعلي.
* الصور أو الأوصاف أو الأسعار أو الحالات أو المدد أو شروط الاستخدام الخاصة بالميزة.
* سجلات التفاعل أو التواصل أو الطلب أو العرض أو الإغلاق أو أي بيانات لازمة لتشغيل الميزة.

الأغراض:

* تمكين المستخدمين من الاستفادة من مزايا التطبيق التجارية أو التفاعلية.
* إدارة البلاغات ومنع الاحتيال.
* تحسين جودة الخدمة وتجربة المستخدم.
* الالتزام بالمتطلبات النظامية أو التشغيلية.

حدود المسؤولية:

أي تعامل أو اتفاق أو دفع أو تسليم أو ضمان أو استرجاع يتم خارج آلية معتمدة وصريحة داخل التطبيق يكون على مسؤولية الأطراف المعنية وحدهم. ولا يتحمل التطبيق مسؤولية جودة المنتجات أو الخدمات أو الأسعار أو صحة الادعاءات أو التزامات الأطراف، إلا بالقدر الذي ينص عليه التطبيق صراحة وبما تسمح به الأنظمة.

### 5.6 بيانات المزايا التنظيمية والاجتماعية

قد تشمل بيانات المزايا التنظيمية والاجتماعية أي بيانات يستخدمها الأعضاء لتنظيم أنشطة الديوانية أو متابعة الحضور أو المواعيد أو المشاركات أو التفاعل أو التنبيهات أو أي وظيفة مشابهة.

الأغراض:

* تنظيم شؤون الديوانية.
* تنبيه الأعضاء بالمستجدات.
* تحسين التنسيق بين الأعضاء.
* عرض مؤشرات تفاعل أو نشاط داخل التطبيق عند توفرها.

ولا تعد أي مؤشرات أو تنبيهات أو تصنيفات أو نتائج داخل التطبيق حكمًا رسميًا أو تقييمًا قانونيًا أو ماليًا أو اجتماعيًا من التطبيق على المستخدم، وإنما هي أدوات تنظيمية أو تفاعلية قابلة للتغيير بحسب سلوك المستخدم داخل التطبيق أو إعدادات الخدمة.

### 5.7 بيانات الإشعارات وآخر النشاطات

قد تشمل:

* تنبيهات الحساب والعضوية.
* تنبيهات المحتوى أو النشاطات داخل الديوانية.
* تنبيهات الخصائص الحالية أو المستقبلية.
* آخر النشاطات داخل الديوانية.

الأغراض:

* إبلاغ المستخدم بالمستجدات المهمة.
* تحسين تجربة المتابعة والتنظيم.
* حفظ تسلسل الأحداث داخل الديوانية.

### 5.8 البيانات التقنية والأمنية

قد تشمل:

* عنوان IP.
* نوع الجهاز.
* نظام التشغيل.
* سجلات الأخطاء.
* سجلات الأداء.
* محاولات التحقق.
* سجلات أمنية وتشغيلية.

الأغراض:

* حماية التطبيق.
* اكتشاف الأعطال.
* تحسين الأداء.
* منع إساءة الاستخدام.
* دعم التحقيق في الحوادث الأمنية.

* نعم، داخل سياق الديوانية أو الخاصية المستخدمة.

الأغراض:

* App Functionality.

يحتفظ التطبيق بحقه في حذف أو تقييد أو تعليق أي محتوى أو حساب أو تعامل أو ميزة عند وجود مخالفة أو بلاغ أو اشتباه احتيال أو طلب نظامي صحيح.

## 12. Store Listing Compliance Checklist

قبل رفع التطبيق للمتاجر، يجب التأكد من التالي:

### قانوني وخصوصية

* سياسة الخصوصية موجودة داخل التطبيق.
* شروط الاستخدام موجودة داخل التطبيق.
* سياسة الاستخدام والمحتوى موجودة داخل التطبيق.
* سياسة الاحتفاظ والحذف موجودة داخل التطبيق.
* حذف الحساب متاح داخل التطبيق.
* صفحة حذف الحساب متاحة على الموقع.
* بريد التواصل الرسمي يعمل: [info@diwaniya.online](mailto:info@diwaniya.online).
* الإفصاحات تغطي البيانات والمزايا والإعلانات والتسويق والمشاركة حسب سلوك التطبيق الفعلي.

### تقني وأمني

* بيئة الإنتاج تستخدم HTTPS.
* لا توجد روابط localhost أو IP داخلي في نسخة الإنتاج.
* لا يتم طباعة رموز التحقق أو التوكنات في سجلات الإنتاج.
* تخزين التوكنات آمن.
* قيود محاولات التحقق مفعلة.
* صلاحيات الأعضاء والمدراء محمية من الباكند.
* حذف العضو أو الحساب محمي بشرط الرصيد أو الالتزام القائم.
* migrations محدثة في بيئة الإنتاج.
* بيانات مزايا التطبيق تخضع لقواعد الصلاحيات والعزل.

### تجربة المستخدم

* شاشة الخصوصية والشروط واضحة.
* شاشة حذف الحساب واضحة.
* رسائل الأخطاء مفهومة.
* لا يتم حذف أو تسجيل خروج المستخدم عند فشل الطلب.
* الإشعارات لا تظهر مكررة بشكل مزعج.
* السجلات أو المؤشرات المهمة تظهر دون الحاجة لفتح شاشة أخرى عند الحاجة.
* أي ميزة جديدة تعرض ضوابطها بوضوح عند الحاجة.

### المتجر

* وصف التطبيق لا يعد بخدمات دفع أو ضمان مالي إلا إذا تم تفعيلها رسميًا.
* لقطات الشاشة لا تعرض بيانات حقيقية.
* لا تستخدم صور أشخاص أو أرقام جوال حقيقية.
* التصنيف العمري مناسب لطبيعة التطبيق.
* الإفصاحات في Apple وGoogle متسقة مع سياسة الخصوصية وسلوك التطبيق.

## 14. قرارات معتمدة قبل الإطلاق الأول

يعتمد تطبيق ديوانية القرارات التالية:

1. لا يتم بيع البيانات الشخصية المباشرة.
2. يمكن استخدام بيانات عامة أو مجمعة أو غير معرفة لأغراض تشغيلية أو تحليلية أو تسويقية وفق الأنظمة والسياسات.
3. يمكن مشاركة بيانات محدودة مع مزودي الخدمة أو الشركاء بالقدر اللازم لتشغيل التطبيق أو تحسينه أو حمايته أو تقديم مزاياه.

7. المؤشرات أو النتائج أو التنبيهات داخل التطبيق أدوات تنظيمية أو تفاعلية وليست أحكامًا رسمية أو قانونية على المستخدمين.
8. البيانات الجماعية داخل الديوانية قد تبقى كسجل جماعي عند حذف الحساب، مع فصلها عن هوية المستخدم عند الحاجة.
9. الرصيد أو الالتزام القائم يمنع حذف العضو أو الحساب إلى حين التسوية أو المعالجة.
10. أي تغيير في البيانات أو الإعلانات أو الدفع أو المشاركة أو المزايا الجوهرية يتطلب مراجعة السياسات وإفصاحات المتاجر قبل التفعيل.

5. أن تكون أي مشاركة للبيانات مع مزودي الخدمة أو الشركاء محددة الغرض ومبررة ومحمية.
6. أن تكون أي استخدامات تسويقية أو إعلانية أو تحليلية مفصحًا عنها ومبنية على أساس نظامي صحيح.
7. أن تتم مراجعة أي ميزة عالية الأثر على الخصوصية أو المستخدمين قبل تفعيلها.

---

## 18. اعتماد الوثيقة

تُعد هذه الوثيقة مرجعًا داخليًا لتوجيه قرارات الامتثال وتجهيز إفصاحات المتاجر ومراجعة المزايا قبل الإطلاق أو التحديثات الجوهرية. ولا تُنشر هذه الوثيقة للمستخدمين بصيغتها الداخلية، وإنما تُستخدم لضمان اتساق السياسات والإفصاحات وسلوك التطبيق الفعلي.

تهدف هذه الوثيقة إلى حماية التطبيق وتشغيله بما يتوافق مع الأنظمة المعمول بها في المملكة العربية السعودية، وسياسات متاجر التطبيقات، وأفضل الممارسات العالمية في الخصوصية وحوكمة البيانات. ولا تغني هذه الوثيقة عن المراجعة القانونية النهائية قبل الإطلاق التجاري الرسمي أو قبل تفعيل أي معالجة عالية الأثر للبيانات.

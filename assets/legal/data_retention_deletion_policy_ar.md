\## 1. مقدمة



تهدف هذه السياسة إلى توضيح آلية احتفاظ تطبيق \*\*ديوانية\*\* بالبيانات، وطرق حذفها أو تقييدها أو فصلها عن هوية المستخدم عند الحاجة، بما يحقق التوازن بين خصوصية المستخدمين، حماية حقوق أعضاء الديوانية، سلامة السجلات المشتركة، ومتطلبات التشغيل والأمن والامتثال النظامي.



تُعد هذه السياسة مكملة لسياسة الخصوصية وشروط الاستخدام وسياسة الاستخدام والمحتوى، وتُستخدم كمرجع داخلي وتنفيذي عند بناء أو تشغيل خصائص مثل حذف الحساب، حذف الديوانية، حذف الصور والملفات، إدارة المصاريف المشتركة، وسجلات الإدارة والتدقيق.

\## 2. المبادئ العامة للاحتفاظ والحذف



يعتمد التطبيق في تعامله مع البيانات على المبادئ التالية:



1\. الاحتفاظ بالبيانات فقط للمدة اللازمة لتقديم الخدمة أو حماية المستخدمين أو الامتثال للمتطلبات النظامية أو التشغيلية.

2\. حذف البيانات أو تعطيلها أو فصلها عن هوية المستخدم عندما لا تعود هناك حاجة مشروعة للاحتفاظ بها.

3\. عدم حذف البيانات بطريقة تضر بحقوق أعضاء آخرين أو تؤثر على سلامة سجل جماعي داخل الديوانية.

4\. تقييد أو تأجيل حذف بعض البيانات عند وجود رصيد قائم، نزاع، بلاغ، خطر أمني، أو طلب نظامي صحيح.

5\. الاحتفاظ بسجلات إدارية أو أمنية محدودة عند الحاجة لأغراض الحوكمة، الحماية، أو إثبات تنفيذ إجراء معين.

6\. فصل البيانات الشخصية عن المحتوى الجماعي متى كان ذلك أنسب من الحذف الكامل لحماية حقوق الأطراف الأخرى.

7\. تطبيق مبدأ الحد الأدنى من البيانات عند مشاركة أو تخزين أو أرشفة أي بيانات.

\## 3. أنواع البيانات المشمولة



تشمل هذه السياسة البيانات التي يعالجها التطبيق، ومن ذلك:



\* بيانات الحساب والتحقق.

\* بيانات الديوانية والعضوية والصلاحيات.

\* طلبات الانضمام.

\* المحادثات والرسائل والمرفقات.

\* الصور والألبوم والملفات والصوتيات.

\* المقاضي والاحتياجات المشتركة.

\* المصاريف المشتركة والأرصدة والتسويات.

\* التصويتات.

\* الإشعارات وآخر النشاطات.

\* السوق والعروض والقوائم والمزادات عند توفرها.

\* سجلات الدعم والإدارة والتدقيق.

\* السجلات التقنية والأمنية.

\## 4. مدد الاحتفاظ المبدئية



تختلف مدة الاحتفاظ بحسب نوع البيانات والغرض من استخدامها. والمدد التالية مبدئية وقابلة للتعديل بحسب المتطلبات النظامية أو التشغيلية أو الأمنية أو طبيعة الخدمة.



\### 4.1 بيانات الحساب



يتم الاحتفاظ ببيانات الحساب طوال مدة بقاء الحساب قائمًا. وعند طلب حذف الحساب، يتم حذف أو تعطيل أو فصل البيانات الشخصية عن السجلات المرتبطة بها، ما لم توجد حاجة مشروعة للاحتفاظ ببعض البيانات لفترة محدودة.



\### 4.2 رموز التحقق OTP



يتم الاحتفاظ برموز التحقق ومحاولات التحقق لمدة قصيرة جدًا بحسب صلاحية الرمز ومتطلبات الحماية من الإساءة. ويتم إبطال الرمز بعد انتهاء صلاحيته أو استخدامه أو تجاوز عدد المحاولات المسموح بها.



\### 4.3 بيانات الديوانية



يتم الاحتفاظ ببيانات الديوانية طوال مدة بقاء الديوانية قائمة. وعند حذف الديوانية، يتم حذف محتواها النشط من النظام بحسب الإمكانات الفنية وسياسة الاحتفاظ، مع إمكانية الاحتفاظ بسجل إداري محدود يثبت وقوع الحذف دون الاحتفاظ بالمحتوى التفصيلي.



\### 4.4 بيانات العضوية والصلاحيات



يتم الاحتفاظ ببيانات العضوية طوال مدة ارتباط المستخدم بالديوانية. وقد يتم الاحتفاظ بسجل محدود لبعض تغييرات الأدوار أو الإزالة أو الانضمام لأغراض الحوكمة أو حماية الحقوق أو مراجعة البلاغات.



\### 4.5 طلبات الانضمام



يتم الاحتفاظ بطلبات الانضمام لمدة محدودة بعد قبولها أو رفضها لأغراض السجل وتجربة المستخدم والحوكمة الداخلية. وقد يتم حذفها أو أرشفتها أو فصلها عن بعض البيانات الشخصية عند انتهاء الحاجة إليها.



\### 4.6 المحادثات والرسائل



يتم الاحتفاظ بالرسائل داخل الديوانية طالما بقيت الديوانية قائمة أو إلى حين حذف الرسالة أو حذف الديوانية أو تطبيق سياسة احتفاظ أخرى. وقد يتم الاحتفاظ المؤقت ببعض السجلات عند وجود بلاغ، نزاع، مخالفة، أو طلب نظامي صحيح.



\### 4.7 الصور والملفات والصوتيات



يتم الاحتفاظ بالصور والملفات والصوتيات طالما بقيت داخل الديوانية أو إلى حين حذفها من المستخدم المخول أو حذف الديوانية. وقد يتم الاحتفاظ المؤقت بنسخ أو سجلات محدودة عند الحاجة الأمنية أو التشغيلية أو النظامية.



\### 4.8 المقاضي



يتم الاحتفاظ ببيانات المقاضي طوال مدة الحاجة إليها داخل الديوانية. ويمكن حذفها أو تعديلها بحسب صلاحيات المستخدمين ووظائف التطبيق.



\### 4.9 المصاريف المشتركة والأرصدة



يتم الاحتفاظ ببيانات المصاريف المشتركة طوال مدة الحاجة إلى سجل المصاريف، وقد تطول مدة الاحتفاظ بها عند وجود أرصدة قائمة، تسويات غير مقفلة، نزاعات، أو حاجة لحماية حقوق الأعضاء وسلامة السجل.



لا يجوز حذف أو إزالة أثر عضو من سجل المصاريف بطريقة تؤثر على حقوق الأعضاء أو تجعل السجل غير مفهوم أو غير قابل للمراجعة.



\### 4.10 التصويتات



يتم الاحتفاظ ببيانات التصويتات طوال مدة الحاجة إليها داخل الديوانية أو إلى حين إغلاقها أو حذفها أو أرشفتها بحسب وظائف التطبيق. وقد يتم الاحتفاظ بسجل محدود عند الحاجة التنظيمية أو لحفظ تسلسل النشاط داخل الديوانية.



\### 4.11 الإشعارات وآخر النشاطات



يتم الاحتفاظ بالإشعارات وآخر النشاطات لمدة محدودة لتحسين تجربة المستخدم وعرض السجل داخل التطبيق. وقد يتم حذفها أو أرشفتها بعد انتهاء الحاجة إليها.



\### 4.12 السوق والعروض والقوائم والمزادات



يتم الاحتفاظ ببيانات السوق أو العروض أو القوائم أو المزادات طوال مدة عرضها أو الحاجة إليها، وقد يتم الاحتفاظ بسجلات محدودة بعد إغلاقها أو حذفها لأغراض الحماية، البلاغات، منع الاحتيال، أو معالجة النزاعات.



\### 4.13 سجلات الدعم والإدارة والتدقيق



قد يتم الاحتفاظ بسجلات الدعم والإدارة والتدقيق لمدة أطول من البيانات التشغيلية العادية، وذلك لأغراض الحوكمة، الأمن، حماية الحقوق، متابعة البلاغات، وتحسين الخدمة.



\### 4.14 السجلات التقنية والأمنية



يتم الاحتفاظ بالسجلات التقنية والأمنية مبدئيًا لمدة تصل إلى 90 يومًا، وقد تختلف المدة بحسب الحاجة الأمنية أو التشغيلية أو متطلبات التحقيق في الحوادث أو منع إساءة الاستخدام.

\## 5. حذف الحساب



يحق للمستخدم طلب حذف حسابه وفق الآلية التي يوفرها التطبيق أو عبر التواصل مع البريد الرسمي.



عند حذف الحساب، قد يتخذ التطبيق واحدًا أو أكثر من الإجراءات التالية بحسب نوع البيانات وطبيعتها:



1\. حذف بيانات الحساب الشخصية من النظام النشط.

2\. تعطيل الحساب ومنع استخدامه مستقبلًا.

3\. فصل هوية المستخدم عن بعض السجلات الجماعية التي لا يمكن حذفها دون التأثير على حقوق أعضاء آخرين.

4\. الاحتفاظ بسجل محدود لطلب الحذف أو تنفيذه لأغراض الحوكمة أو الامتثال أو الحماية.

5\. تقييد الحذف أو تأجيل بعض آثاره عند وجود رصيد قائم، نزاع، مخالفة، بلاغ، أو طلب نظامي صحيح.



لا يعني حذف الحساب بالضرورة حذف كل محتوى جماعي سبق أن أضافه المستخدم داخل ديوانية إذا كان ذلك المحتوى جزءًا من سجل مشترك أو مرتبطًا بحقوق أعضاء آخرين أو بسجل مصاريف أو نشاط جماعي.



\---



\## 6. حذف الديوانية



عند حذف الديوانية من قبل المستخدم المخول، يتم التعامل مع بياناتها وفق المبادئ التالية:



1\. حذف المحتوى النشط المرتبط بالديوانية بحسب الإمكانات الفنية.

2\. حذف أو تعطيل العضويات المرتبطة بها.

3\. حذف أو أرشفة المقاضي والتصويتات والإشعارات والنشاطات المرتبطة بها بحسب الحاجة.

4\. حذف الصور والملفات أو نقلها إلى حالة حذف/أرشفة بحسب البنية التقنية المعتمدة.

5\. الاحتفاظ بسجل إداري محدود يثبت أن الديوانية حُذفت، دون الاحتفاظ بالمحتوى التفصيلي إلا عند وجود سبب مشروع.



إذا كانت هناك أرصدة أو مصاريف أو تسويات قائمة، فقد يتم تقييد حذف الديوانية أو طلب معالجة تلك الالتزامات أولًا، أو الاحتفاظ بسجلات محدودة لحماية حقوق الأعضاء.



\---



\## 7. حذف العضو من الديوانية



يمكن للمؤسس أو المدير إزالة عضو من الديوانية وفق الصلاحيات والقيود المعتمدة.



ومع ذلك، لا يمكن إزالة عضو إذا كان له أو عليه رصيد قائم أو كان مرتبطًا بتسويات غير مقفلة أو مصاريف تؤثر على سجل الديوانية. ويجب تسوية الرصيد أو إغلاق الالتزامات أولًا قبل إتمام الإزالة.



عند إزالة العضو، قد يتم:



\* إلغاء وصوله إلى الديوانية ومحتواها المستقبلي.

\* الاحتفاظ ببعض السجلات السابقة التي أضافها داخل الديوانية إذا كانت جزءًا من سجل جماعي.

\* إظهار اسمه أو معرفه السابق في بعض السجلات بقدر ما يلزم لفهم السجل أو حماية حقوق الأعضاء.

\* تقييد أو إخفاء بعض البيانات الشخصية بحسب ما تسمح به وظائف التطبيق والسياسات المعتمدة.

\## 8. حذف المحتوى داخل الديوانية



\### 8.1 الرسائل



قد يتيح التطبيق للمستخدم حذف رسائله أو بعض محتواه بحسب الوظائف المتاحة. وقد تبقى آثار محدودة أو سجلات فنية عند الحاجة للأمن أو الحوكمة أو معالجة البلاغات.



\### 8.2 الصور والملفات



يمكن للعضو حذف الصور أو الملفات التي أضافها بنفسه إذا كانت الخاصية تسمح بذلك. ويجوز للمؤسس أو المدير حذف الصور أو الملفات المخالفة أو غير المناسبة وفق الصلاحيات المتاحة.



\### 8.3 المقاضي والتصويتات



يمكن حذف أو تعديل المقاضي أو التصويتات بحسب صلاحيات المستخدم ووظائف التطبيق. وقد يتم الاحتفاظ بسجل محدود عند الحاجة التنظيمية أو الأمنية أو لحفظ تسلسل النشاط.



\### 8.4 السوق والعروض والقوائم



يمكن حذف أو إغلاق العروض أو القوائم أو المزادات بحسب الصلاحيات والضوابط المعتمدة. وقد يحتفظ التطبيق بسجل محدود عند الحاجة لمنع الاحتيال أو معالجة البلاغات أو حماية المستخدمين.

\## 10. النسخ الاحتياطية والأرشفة



قد تحتفظ الأنظمة التقنية بنسخ احتياطية أو سجلات أرشيفية لفترة محدودة لأغراض استمرارية الخدمة، الاستعادة من الأعطال، الأمن، أو الامتثال.



قد لا يتم حذف البيانات فورًا من جميع النسخ الاحتياطية عند حذفها من النظام النشط، لكنها تُزال أو تُستبدل وفق دورة الاحتفاظ بالنسخ الاحتياطية المعتمدة، ما لم توجد حاجة نظامية أو أمنية للاحتفاظ بها لفترة أطول.

\## 11. التقييد بدل الحذف



في بعض الحالات، قد يكون التقييد أو الفصل عن الهوية أو التعطيل أنسب من الحذف الكامل، خصوصًا عندما تكون البيانات:



\* مرتبطة بسجل جماعي داخل الديوانية.

\* مرتبطة بمصاريف أو أرصدة أو تسويات.

\* لازمة لحماية حقوق أعضاء آخرين.

\* لازمة للتحقيق في بلاغ أو مخالفة.

\* مطلوبة للامتثال أو الحماية أو الأمن.



في هذه الحالات، قد يتم تقييد الوصول إلى البيانات أو إخفاء بعض عناصرها أو فصلها عن هوية المستخدم بدل حذفها بالكامل.
